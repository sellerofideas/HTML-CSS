Entendendo as caixas em HTML.

Todo conteúdo é criado em forma de caixas, enxergar a página de acordo com essa perspectiva ajuda e muito na hora de criar. Cada caixa por padrão é criada apenas com o conteúdo, a única exceção é o body criado com uma margin de 8px. Podemos acrescentar em cada elemento, valores para a margin, padding, border e outline.

Propriedades das caixas

  • Conteúdo

    • Todo conteúdo (content) possui um tamanho de caixa chamado box-size composto de altura (height) e largura (width).

  • Padding

    • Fora da área de conteúdo temos a área de preenchimento (padding). O padding de fica da borda para dentro (border).

  • Border

    • É a área da borda do conteúdo. Cada borda pode ser configurada com uma cor, espessura e formato.

  • Outline

    • O espaço entre a margem e a borda onde podemos determinar o contorno (outline) é muito pouco utilizado. Ele é um traçado visual que podemos criar fora da borda e o cálculo da sua espessura faz parte da margem estabelecida.

  • Margin

    • A margem é a parte exterior que fica depois da borda.

A nova propriedade box-sizing define as dimensões height e width a partir da borda (border-box) e não apenas a partir do conteúdo (content-box).

A medida total da height e width que essa caixa vai ocupar é feita pela soma de todas as medidas, margin, padding, border e content.

Box inline e box em bloco

Partindo do princípio de que todo conteúdo em CSS é representado por caixas temos duas categorias de caixas.

O shorthand das medidas do border e do outline são:

  • border: tamanho tipo cor;
  • outline: tamanho tipo cor;

A margin e o padding são configuradas com um valor para os quatro lados, um valor para direita e esquerda, outro para em cima e em baixo, e também um valor para cada lado em sentido horário, cima, direita, baixo, esquerda

Outra forma de configuração é colocando auto e deixando com que o navegador calcule as margens deixando o bloco sempre no meio.

Box inline

O box inline é usado quando queremos trabalhar com uma caixa no conteúdo. Sua característica é ocupar somente o espaço que lhe cabe sem precisar pular linha para ocupar uma linha inteira.

Exemplos de bloco in line:

  • <a> (ou elemento âncora), com o atributo href cria-se um hiperligação nas páginas web, arquivos, endereços de emails, ligações na mesma página ou endereços na URL. O conteúdo dentro de cada precisará indicar o destino do link.

  • <abbr> (ou Elemento de Abreviação HTML) representa uma abreviação e opcionalmente fornece uma descrição completa para ela.

  • <b> representa um intervalo de texto estilísticamente diferente do texto normal, sem transmitir qualquer importância ou relevância.

  • <bdo> é usado para substituir a direcionalidade atual do texto, ou seja, inverter as palavras. Isso faz com que a direcionalidade dos personagens seja ignorada em favor da direcionalidade especificada.

  • <br>produz uma quebra de linha em um texto (carriage-return).É útil para escrever poemas ou um endereço, onde a divisão de linha é significante.

  • <buttom> representa um botão clicável.

  • <cite> representa uma referência a um trabalho artístico.

  • <code> apresenta seu conteúdo estilizado de maneira a indicar que o texto é um pequeno fragmento de código.

  • <dfn> representa uma instância de definição de um termo.

  • <em> marca o texto que tem ênfase. O elemento em pode ser aninhado, com cada nível de aninhamento indicando um grau maior de ênfase

  • <img> representa a inserção de imagem no documento, sendo implementado também pelo HTML5 para uma melhor experiência com o elemento figure e figcaption.

  • <imput> é usado para criar controles interativos para formulários baseados na web para receber dados do usuário. A semântica de um input varia consideravelmente dependendo do valor de seu atributo type.

  • <kbd> O elemento representa um intervalo de texto embutido que denota a entrada textual do usuário de um teclado, entrada de voz ou qualquer outro dispositivo de entrada de texto.

  • <label> representa uma legenda para um item em uma interface de usuário. Ele pode estar associado com um elemento de controle, colocando este dentro do elemento label, ou usando o atributo for.

  • <map> é usado com os elementos para definir um mapa de imagem (a área clicável do link).

  • <object> O elemento representa um recurso externo, que pode ser tratado como uma imagem, um contexto de navegação aninhado ou um recurso a ser tratado por um plug-in.
  • <output> é um elemento de contêiner no qual um site ou aplicativo pode injetar os resultados de um cálculo ou o resultado de uma ação do usuário.

  • <q> indica que o texto dentro da tag é uma pequena citação. Este elemento destina-se a citações curtas que não requerem marcações de parágrafo; para citações maiores use o elemento blockquote.

  • <samp> é usado para incluir texto embutido que representa a saída de amostra (ou cota) de um programa de computador.

  • <script> é usado para incluir ou referenciar um script executável.

  • <select> representa um controle que apresenta um menu de opções. As opções dentro do menu são representadas pelo elemento option, que podem ser agrupados por elementos optgroup. As opções podem ser pré-selecionadas para o usuário.

  • <small> representa comentários laterais e letras pequenas, como direitos autorais e texto legal, independentemente de sua apresentação estilizada.

  • <span> é um conteiner generico em linha para conteúdo fraseado , que não representa nada por natureza. Ele pode ser usado para agrupar elementos para fins de estilo (usando os atributos class ou id ), ou para compartilhar valores de atributos como lang.

  • <strong> dá ao texto uma forte importância, e é tipicamente mostrado em negrito.

  • <sub> especifica o texto embutido que deve ser exibido como subscrito apenas por motivos tipográficos.

  • <textarea> representa um controle de edição para uma caixa de texto, útil quando você quer permitir ao usuário informar um texto extenso em formato livre, como um comentário ou formulário de retorno.

  • <var> representa uma variável em uma expressão matemática ou um contexto de programação.

Box em bloco

O box em bloco, por outro lado, já ocupa uma linha inteira pulando uma linha.

Exemplo de box em bloco

Existem algumas tags in block que talvez você já conheça.

  • <Addres> Pode ser usado em uma variedade de contextos, como fornecer informação de contato de uma empresa no header da página, ou indicar o autor de um artigo incluindo um elemento address dentro do article.

  • <Article> Este poderia ser o post de um fórum, um artigo de revista ou jornal, um post de um blog, um comentário enviado por um usuário, um gadget ou widget interativos, ou qualquer outra forma de conteúdo independente.

  • <aside> Para adicionar um conteúdo secundário ou que tenha a ver com o conteúdo principal. Elas muitas vezes contem explicações laterais, como a definição de um glossário; conteúdo vagamente relacionado, como avisos; a biografia do autor; ou, em aplicações “web”, informações de perfil ou links de “blogs” relacionados.

  • <blockquote>Indica que o texto incluído é uma longa citação.

  • <canvas> Pode ser utilizado para desenhar gráficos utilizando scripts, além de desenhar gráficos, ele pode ser usado para fazer composições de fotos e também para animaçõess

  • <dd> Fornece detalhes ou uma definição mais completa do termo definido por dt numa lista de descrições dl.

  • <dl> Engloba uma lista de pares de termos e descrições. Um uso comum para este elemento é para implementar um glossário ou exibir metadados(uma lista de pares-chave e valor).

  • <dt> Identifica um termo na lista de definição. Este elemento pode ocorrer somente em um elemento filho de. Geralmente seguido por um elemento dd.

  • <div> É um contêiner genérico para conteúdo de fluxo, que de certa forma não representa nada. Ele pode ser utilizado para agrupar elementos para fins de estilos (usando "class" ou "id"), porque eles compartilham valores de atributos, como "lang". Ele deve ser utilizado somente quando não tiver outro elemento de semântica tal como article ou nav.

  • <fieldset> É usado para agrupar elementos, assim como "labels" dentro de um formulário, “web”.

  • <figcaption> Representa uma legenda ou uma legenda associada com uma figura, ou ilustração descrita pelo resto dos dados do elemento <figure> que seu elemento pai.

  • <figure> Representa o conteúdo independente, frequentemente com uma legenda (figcaption), e é normalmente referido como uma única unidade.

  • <footer> Representa um rodapé para o seu sectioning content (conteúdo de seção) mais próximo ou sectioning root elemento, ou seja, seu parente mais próximo, article, aside, nav, section, blockquote, body, details, fieldset, figure, td. Normalmente um rodapé contém informações sobre o autor da seção de dados, direitos autorais ou links para documentos relacionados.

  • <form> Representa uma seção de um documento que contém controles interativos que permitem ao usuário submeter informação a um determinado servidor “web”.

  • <h1-h6> Representam seis níveis de título de seção. H1 É o nível de seção mais alto e h6 é o mais baixo.

  • <header> Representa um grupo de suporte introdutório ou navegacional. Pode conter alguns elementos de cabeçalho mas também outros elementos como um logo, seções de cabeçalho, formulário de pesquisa, e outros.

  • <hr> Representa uma quebra temática entre elementos de nível de parágrafo (por exemplo, uma mudança da cena de uma história, ou uma mudança de tema com uma seção). Nas versões anteriores do HTML, representava uma linha horizontal. Pode continuar sendo exibida como uma linha horizontal nos navegadores, mas agora está definida em termos semânticos, em vez de termos de apresentação.

  • <main> Define o conteúdo principal dentro do <body> em seu documento ou aplicação. Entende-se como conteúdo principal aquele relacionado diretamente com o tópico central da página ou com a funcionalidade central da aplicação.

  • <nav> Para o navegador ou menu. Representa uma seção de uma página que aponta para outras páginas ou para outras áreas da página, ou seja, uma seção com links de navegação.

  • <noscript> Define uma seção de HTML a ser inserida se uma categoria de script não é suportado pela página ou se o script está desativado no navegador.

  • <ol> Para criar uma lista organizada com hierarquia.

  • <p> para criar parágrafo.

  • <pre> É a tag utilizada para representar texto pré-formatado. Um texto dentro desse elemento é tipicamente exibido em uma fonte não proporcional da mesma maneira em que o texto original foi disposto no arquivo. Espaços em branco são mantidos no texto da mesma forma em que este foi digitado.

  • <section> Representa uma seção genérica contida em um documento HTML, geralmente com um título, quando não existir um elemento semântico mais específico para representá-lo.

  • <table> Para a criação de tabelas. Representa dados em duas dimensões ou mais.

  • <tfoot> É um elemento HTML que define um conjunto de linhas as quais farão parte do rodapé de uma tabela HTML.

  • <video> O elemento HTML video é utilizado para incorporar conteúdo de vídeo em um documento HTML ou XHTML.

Na tag "style" dentro da head ou na folha de CSS podemos alterar as caixas dos conteúdos. Para transformar um box inline em um box em bloco basta adicionar o atributo display: inline-block;, também podemos configurar o bloco como inline.

Uma dica é usar o DevTools do navegador para alterar a página e mostrar em tempo real para o cliente como as mudanças ficam no site, caso o cliente goste do resultado é só implementar no código original.

Grouping tags e Semantic tags

Existem duas tags de agrupamento genérico, adiv, uma block-level (box em bloco) e a span uma line-level (box em linha) com a utilidade de unir vários elementos. Com o surgimento do HTML 5 e o conceito de HTML semântico foram criadas tags que servem esse propósito, mas agora dando significado ao código.

Vejamos quais são as semantic tags:

Header

É o cabeçalho da página onde está o menu, mas também pode ser o cabeçalho de uma seção ou artigo.

Nav

Onde estão os links de navegação (o menu).

Main

É o agrupador onde fica o conteúdo principal com seções, artigos e conteúdos periféricos.

Section

É um agrupamento temático de conteúdo que divide o conteúdo da página em porções menores e costuma ter um cabeçalho.

Article

É um conteúdo independente como um post de um blog, uma notícia, etc. e diz respeito ao conteúdo principal.

Aside

Um conteúdo periférico e complementar que normalmente está ao lado do conteúdo principal.

footer

O rodapé do site, seção ou artigo e possui informações de contato, etc.

Sombras das caixas

Sombra é aquilo que dá o contorno dando a impressão de que existe um objeto ali.

Criando sombras

Para criarmos sombras usamos a propriedade box-shadow indicando seu deslocamento, embaçamento e cor. É muito comum usarmos sua shorthand como:box-shadow: 3px 5px 4px black;, nesse caso indicamos primeiro o valor de seu deslocamento horizontal para (h-offset), depois o deslocamento vertical (v-offset), o embaçamento ou espalhamento (blur) e a cor que geralmente é black.

Bordas aredondadas

Podemos trabalhar as bordas e vértices com atributo border-radius, com ele podemos arredondar os vértices das caixas, caso tenham bordas.

    Para arredondar todas de uma vez:
  • border-radius: 10px;
    • Para arredondar as quatro (vértices) com valores diferentes:
  • border-radius: 10px 20px 30px 40px;
    • Para arrendondamento intercalado:
  • border-radius: 20px 30px;
